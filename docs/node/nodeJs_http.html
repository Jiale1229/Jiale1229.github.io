http 服务器模块实现数据交互

http 服务器模块实现数据交互

🙆‍♂️ 徐乐乐 🗓️ 2022/10/21 🏷️ NodeJs

前言

http 模块主要用于搭建 HTTP 服务端和客户端 客户端，一般指浏览器，浏览器可以通过 HTTP 协议向服务器请求数据。 服务端，一般指 Web 服务器，可以接收客户端请求，并向客户端发送响应数据。

# http 基本架构

const http = require('http');
var fs = require('fs');
var url = require('url');

// 创建服务器
http.createServer((req, res) => {
    // 解析请求，包括文件名
    let pathname = url.parse(req.url).pathname;
    let postfix = pathname.match(/(.[^.]+|)$/)[0];//取得后缀名
    // 从文件系统中读取请求的文件内容,注意设置相应的请求头
    fs.readFile(__dirname + pathname,(err, data)=> {
        if (err) {
            console.log(err);
            res.writeHead(404, {'Content-Type': 'text/html;charset=UTF8'});
        } else {
            // HTTP 状态码: 200 : OK
            if (postfix === 'html') {
               //为了防止中文显示乱码的问题，需要设置响应头 Content-Type 的值 为 text/html;charset = utf-8
                res.writeHead(200, {'Content-Type': 'text/html;charset=UTF8'});
            } else if (postfix === 'css') {
                res.writeHead(200, {'Content-Type': 'text/css'});
            } else if (postfix === 'js') {
                res.writeHead(200, {'Content-Type': 'application/javascript'});
            }else {
								...
            }
            // 响应文件内容
            res.write(data.toString());
        }
        
        //  发送响应数据
         
        res.end();
    });
}).listen(8080)

  • req： 服务器解析用户提交的http请求报文，将结果解析到request对象中，凡是要获取和用户请求的数据都可以通过request对象获取。常见

    • req.method 请求方法 GET/POST
    • req.url 请求路径

  • res： 在服务器端用来向用户做出响应的对象，凡是需要向用户（客户端）响应操作，都需要通过response对象来进行。常见

    • res.writeHead 设置响应头 必须在write()和end()方法前调用

  • res.write 向前端返回数据

    • res.end 每次响应都必须调用该方法，用来结束响应

注意

对于不同的文件类型，比如html,css,js,请求头里面的文件类型需要根据不同的文件类型注明

文件的路径需要表达准确，fs.readFile方法的第一个参数path是已起服务的文件为根目录

# url.parse(urlString,boolean,boolean)

将一个url的字符串解析并返回一个url的对象，三个参数

  • urlString：要解析成对象的url字符串
  • parseQueryString：是否把url中 ? 之后的查询参数解析成 query 对象，默认为false
  • slashesDenoteHost：当不知道url协议时，是否以 // 为依据识别host，默认为false
const url = require('url');
url.parse('http://localhost:8081/user?name=xulele&age=18#hash', true);

// 得到的结果如下
Url {
    protocol: 'http:', //使用协议
    slashes: true, // (协议的“：”号后是否有“/”)(slashes意思是斜杠)
    auth: null,  // 验证信息
    host: 'localhost:8081', // host主机名
    port: '8081', // 端口号
    hostname: 'localhost', // 不带端口号的主机名
    hash: '#hash', // 哈希值(URL属性中在井号“#”后面的分段)
    search: '?name=xulele&age=18', // 查询字符串(URL属性中跟在问号后面的部分包括?号)
    query:[Object: null prototype]  {name: 'xulele', age: '18'}, // 加true时为object
    // query:'name=xulele&age=18', // 不加true时
    pathname: '/user',  //主机上的一个目录或文件地址。
    path: '/user?name=xulele&age=18',// 带查询的路径名
    href: 'http://localhost:8081/user?name=xulele&age=18#hash'// 原字符串本身
}

# url.format(urlObj)

与parse方法相反，它用于将传入的URL对象生成URL字符串返回

# url.resolve(from,to)

为URL或 href 插入 或 替换原有的标签，

返回结果去除了参数和锚点

  • from: 源地址
  • to: 需要添加或替换的标签
    • '/add' - 将替换基本URL域之后的整个路径。
    • 'add' - 将替换基本URL路径中 /之后的最后一个单词。
url.resolve('http://example.com/one/two', '/add'), // http://example.com/add
url.resolve('http://example.com/one/two', 'add'),  //  http://example.com/one/add
// 带参数 url
url.resolve('http://example.com/one/two/user?name=xulele&age=18#hash', '/add'); // http://example.com/add
url.resolve('http://example.com/one/two/user?name=xulele&age=18#hash', 'add'); // http://example.com/one/two/add

# GET

由于GET请求直接被嵌入在路径中，URL是完整的请求路径，包括了?后面的部分，因此可以手动解析后面的内容作为GET请求的参数。

const http = require('http');
const url = require("url");

http.createServer((req,res)=>{
    let urlStr = url.parse(req.url, true).query;
    ... // 处理数据
})

# Post

POST 请求的内容全部都在请求体中

const http = require('http');
const querystring = require('querystring');

http.createServer((req, res) => {
    // 定义了一个post变量，用于暂存请求体的信息
    let post = '';
    // data 当有数据可读时触发，每当接受到请求体的数据，就累加到post变量中
    req.on('data', function (chunk) {
        post += chunk;
    });
    // end 没有更多数据可读时触发，通过querystring.parse将post解析为真正的POST请求格式 
    req.on('end', function () {
        post = querystring.parse(post);
      	... // 处理数据
        res.end();
    });
})

# 常见的 HTTP 状态码

  • 200 - 请求成功
  • 301 - 资源（网页等）被永久转移到其它URL
  • 404 - 请求的资源（网页等）不存在
  • 500 - 内部服务器错误
